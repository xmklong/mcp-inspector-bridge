## 用户输入
$ARGUMENTS

## 执行指令

1.  **阅读**: 加载当前会话中最近创建的 `specs/[feature_name]/spec.md`。
2.  **起草计划**:
    * 仅创建/更新**一个文件**: `specs/[feature_name]/plan.md`。
    * **语言要求**: 整个输出及其文件均必须使用**简体中文 (Simplified Chinese)**撰写。
    * **第一部分: 架构设计 (Architecture)**:
      - 使用表格列出所有涉及文件、所属层级、改动性质。
      - 简述数据模型/架构影响（如无变更则明确说明）。
      - 如有必要，用文本图或 Mermaid 图展示关键流程。
    * **第二部分: 分步实施 (Step-by-Step)**:
      - 按阶段分组（如"阶段 A: 代码修改"、"阶段 B: 编译验证"、"阶段 C: 文档更新"）。
      - 直接在此文件内列出所有的实施步骤。
3.  **格式化**:
    * 为每一个步骤使用复选框 `- [ ]`，以便它们是可点击的。
    * 标记对应的步骤属于 `[Frontend]`（前端）还是 `[Backend]`（后端）。
4.  **强制步骤（必须包含）**:
    * 必须包含一个 `npm run build` 编译验证步骤。
    * 必须包含一个更新 `UPDATE_LOG.md` 的文档步骤。
5.  **约束**: **严禁**创建 `tasks.md`，请将所有任务都保留在 `plan.md` 中。
