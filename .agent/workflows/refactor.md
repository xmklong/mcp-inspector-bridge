## 用户输入
$ARGUMENTS

## 执行指令

1.  **审计 (Audit)**: 扫描目标目录/文件 `$ARGUMENTS`，分析代码质量问题。
2.  **章程核查 (Constitution Check)**: 对照 `memory/constitution.md` 检查是否存在违规项，所有反馈**必须使用简体中文**。
3.  **问题清单**: 列出发现的所有问题（死代码、过时模式、命名不规范、缺失注释等），按严重程度排序。
4.  **重构 (Refactor)**: 在**不改变业务逻辑**的前提下执行清理：
    * 例如：将 Promises 转换为 Async/Await、提取重复代码、补充 JSDoc 注释等。
    * 每次修改后说明改动理由。
5.  **编译验证**: 执行 `npm run build` 确保重构未引入编译错误。
6.  **验证 (Verify)**: 总结所有改动，确认无功能退化。
