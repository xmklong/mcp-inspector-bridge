## ⚠️ 验证标准（开始前必须先读）

验证报告必须同时满足以下指标：

| # | 检查项 | 合格标准 |
|---|--------|----------|
| 1 | 环境确认 | 明确记录 `npm run build` 是否成功、`dist/` 是否为最新 |
| 2 | 视觉审计 | 逐条对比 spec.md 的视觉需求，每条给出 ✅/❌ 判定 |
| 3 | 边界测试 | 逐条对比 spec.md 的边界情况，每条给出测试结果 |
| 4 | 回归检查 | 确认无已有功能退化 |
| 5 | 最终结论 | 明确给出 **通过 / 不通过** 的总判定 |

## 验证指令

### 1. 环境确认
* 执行 `npm run build`，确认编译通过且 `dist/` 目录产物为最新。
* 提醒用户在 Cocos Creator 编辑器中重新加载插件（开发者 → 重新加载）。
* **记录编译结果**：通过/失败 + 时间戳。

### 2. 视觉审计 (Visual Audit)

* 加载 `specs/[feature_name]/spec.md` 的 **视觉需求** 章节。
* 请求用户在编辑器中操作新功能/修复项并**提供截图**。
* **逐条对比**，输出结果表格：

```
| # | 视觉需求描述 | 判定 | 差异说明 |
|---|-------------|------|---------|
| 1 | 分辨率选择框最小宽度 120px | ✅ 符合 | — |
| 2 | 窄模式下自动换行为两行 | ❌ 不符 | 仍为单行显示 |
```

* 如有截图，嵌入报告中与 spec 描述对照。

### 3. 逻辑审计 (Logic Audit)

* 加载 spec.md 的 **边界情况** 章节。
* **逐条转化为测试用例**，请求用户执行并反馈结果：

```
| # | 边界场景 | 测试步骤 | 期望结果 | 实际结果 | 判定 |
|---|---------|---------|---------|---------|------|
| 1 | 500px 临界值抖动 | 反复拖拽分割线经过 500px | 平滑切换无闪烁 | ? | ✅/❌ |
| 2 | select 宽度保护 | 面板极窄 | 选择框 ≥ 120px | ? | ✅/❌ |
```

* 检查编辑器控制台是否存在新增报错或警告信息。

### 4. 回归检查 (Regression Check)
* 确认修改未破坏原有功能。重点检查：
  - 正常宽度下工具栏布局是否保持不变
  - 按钮功能是否正常（刷新/暂停/FPS/拾取等）
  - 分辨率切换 + 横竖屏切换是否正常工作
  - Webview 游戏画面是否正常显示

### 5. 输出报告

创建 `specs/[feature_name]/verify_report.md`，**必须包含以下结构**：

```markdown
# 验证报告 - [功能名称]

## 测试环境
- 日期：YYYY-MM-DD
- 编译状态：通过/失败
- 编辑器版本：Cocos Creator 2.4.x

## 视觉审计结果
（上述表格）

## 边界测试结果
（上述表格）

## 回归检查结果
- [ ] 正常宽度布局正常
- [ ] 按钮功能正常
- [ ] 分辨率切换正常
- [ ] 游戏画面正常

## 最终结论
**✅ 通过** / **❌ 不通过**（附不通过原因）
```

* **语言要求**: 报告所有文字描述均须使用**简体中文**。

### 6. 汇报

回复用户时，简要总结验证结论：
* 通过了多少项 / 总共多少项
* 如有不通过项，列出最关键的 1-2 个问题
* 建议下一步操作（修复 → 重新验证，或发布）
