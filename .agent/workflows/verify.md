## 验证指令

1.  **环境确认**:
    * 确认 `npm run build` 已成功执行，`dist/` 目录产物为最新。
    * 提醒用户在 Cocos Creator 编辑器中重新加载插件（开发者 → 重新加载）。

2.  **视觉审计 (Visual Audit)**:
    * 请求用户在编辑器中操作新功能/修复项并**提供截图**。
    * 将截图与 `specs/[feature_name]/spec.md` 中定义的视觉需求进行逐条对比。
    * 输出结果: "✅ 符合" 或 "❌ 存在差异 (附具体描述)"。

3.  **逻辑审计 (Logic Audit)**:
    * 根据 `spec.md` 中的边界情况章节，逐条请求用户测试并反馈结果。
    * 检查编辑器控制台是否存在新增报错或警告信息。

4.  **回归检查 (Regression Check)**:
    * 确认修改未破坏原有功能（基于 spec 中列出的回归风险点）。

5.  **输出报告**:
    * 创建 `specs/[feature_name]/verify_report.md` 发布就绪报告。
    * 包含：测试环境、视觉审计结果、逻辑审计结果、回归检查结果、最终结论（通过/不通过）。
    * **语言要求**: 报告所有文字描述均须使用**简体中文**。
