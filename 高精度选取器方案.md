# 运行时高精度选取器 (Inspector Gizmo) 完整落地文档

## 架构核心思想：降维打击
彻底摒弃 Cocos 原生的 `node._hitTest` 与业务场景渲染流。引入主流 3D 引擎的 **Gizmo（辅助线）** 架构：
1. **渲染隔离**：利用“第 31 号幽灵分组”建立独立且全局置顶的 Inspector 摄像机。
2. **拾取反转 (Camera-First)**：废弃“节点找相机”，改为“相机按层级向下发射射线找节点”。
3. **坐标折跃 (Fold-Projection)**：利用两次矩阵投影（业务世界 -> 屏幕 -> 插件世界），实现 100% 免疫缩放/旋转/裁剪的“绝对贴脸”外框绘制。

## 核心模块一：幽灵渲染层初始化 (Gizmo Environment)

此模块负责在运行时（Preview/网页端）动态构建一个绝对干净、置顶、且不干扰业务逻辑的绘制环境。

```typescript
// InspectorCore.ts
const INSPECTOR_GROUP_INDEX = 31; // 霸占最高位作为幽灵分组

export class InspectorCore {
    private inspectorCamera: cc.Camera = null;
    private graphics: cc.Graphics = null;
    private selectedNode: cc.Node = null;
    private targetCamera: cc.Camera = null;

    public init() {
        this.setupGizmoLayer();
        this.hijackUserCameras();
        
        // 建议在引擎的 lateUpdate 中持续绘制，以完美跟随动态移动的节点
        cc.director.on(cc.Director.EVENT_AFTER_UPDATE, this.onLateUpdate, this);
        cc.systemEvent.on(cc.SystemEvent.EventType.TOUCH_END, this.onTouchEnd, this);
    }

    private setupGizmoLayer() {
        // 1. 创建常驻根节点
        const root = new cc.Node('McpInspectorRoot');
        root.groupIndex = INSPECTOR_GROUP_INDEX;
        cc.game.addPersistRootNode(root);

        // 2. 配置专属摄像机
        const camNode = new cc.Node('InspectorCamera');
        camNode.groupIndex = INSPECTOR_GROUP_INDEX;
        root.addChild(camNode);
        
        this.inspectorCamera = camNode.addComponent(cc.Camera);
        this.inspectorCamera.depth = 9999; // 绝对置顶
        // ⚠️ 极其关键：仅清除深度缓存，绝不能清除颜色（否则游戏画面全黑）
        this.inspectorCamera.clearFlags = cc.Camera.ClearFlags.DEPTH;
        this.inspectorCamera.cullingMask = 1 << INSPECTOR_GROUP_INDEX; // 只拍31号分组

        // 3. 配置专属画笔
        const gfxNode = new cc.Node('InspectorGraphics');
        gfxNode.groupIndex = INSPECTOR_GROUP_INDEX;
        root.addChild(gfxNode);
        
        this.graphics = gfxNode.addComponent(cc.Graphics);
        this.graphics.lineWidth = 2;
        this.graphics.strokeColor = cc.Color.RED;
        this.graphics.fillColor = cc.color(255, 0, 0, 50); // 半透明红色填充
    }

    /**
     * ⚠️ 核心黑魔法：剥夺业务相机对 31 号分组的渲染权
     * 必须在运行时动态修补，防止业务相机（默认 cullingMask=-1）拍出双重红框
     */
    private hijackUserCameras() {
        cc.Camera.cameras.forEach(cam => {
            if (cam !== this.inspectorCamera) {
                // 利用位运算 NAND，强制把第 31 位设为 0
                cam.cullingMask &= ~(1 << INSPECTOR_GROUP_INDEX);
            }
        });
    }
}
```

## 核心模块二：Camera-First 空间射线拾取算法

完全绕开 `node._hitTest` 和 `Camera.findCamera` 的缺陷。按摄像机的渲染层级（`depth`）从上往下“发物理穿透”。

```typescript
    // 续 InspectorCore.ts
    private onTouchEnd(event: cc.Event.EventTouch) {
        // getLocation() 返回的是已经对齐 Canvas 的逻辑屏幕坐标，直接可用
        const screenPos = event.getLocation(); 
        
        // 动态修补一下相机（防止用户中途 new 了新相机）
        this.hijackUserCameras();

        const result = this.pickNode(screenPos);
        if (result) {
            this.selectedNode = result.node;
            this.targetCamera = result.camera;
            console.log("选中节点:", this.selectedNode.name, "归属相机:", this.targetCamera.node.name);
        } else {
            this.selectedNode = null;
            this.targetCamera = null;
            this.graphics.clear();
        }
    }

    private pickNode(screenPos: cc.Vec2): { node: cc.Node, camera: cc.Camera } | null {
        // 1. 获取所有业务摄像机，按 depth 降序排列 (UI 顶层相机排在前面)
        const validCameras = cc.Camera.cameras
            .filter(cam => cam !== this.inspectorCamera && cam.node.activeInHierarchy)
            .sort((a, b) => b.depth - a.depth);

        // 2. 逐个摄像机进行射线检测
        for (const camera of validCameras) {
            // 将屏幕坐标转换为该摄像机视口下的世界坐标
            const worldPos = camera.getScreenToWorldPoint(screenPos);
            
            // 传入场景根节点进行 DFS
            const hitNode = this.walkSceneForCamera(cc.director.getScene(), camera, worldPos);
            if (hitNode) {
                return { node: hitNode, camera: camera };
            }
        }
        return null;
    }

    private walkSceneForCamera(node: cc.Node, camera: cc.Camera, worldPos: cc.Vec2): cc.Node | null {
        // 基础不可见裁剪
        if (!node.activeInHierarchy || node.opacity === 0) return null;
        
        // 🛡️ 绝对壁垒：如果不属于这个摄像机渲染，直接剪枝！杜绝跨相机错误命中！
        if ((camera.cullingMask & (1 << node.groupIndex)) === 0) return null;

        // 逆后序遍历子节点 (保证 Z 轴靠前的后渲染节点先被拾取)
        for (let i = node.childrenCount - 1; i >= 0; i--) {
            const hit = this.walkSceneForCamera(node.children[i], camera, worldPos);
            if (hit) return hit;
        }

        // 自身判定 (幽灵节点剥离：必须有有效渲染组件才可被点击)
        if (this.isValidRenderNode(node)) {
            // 采用引擎最底层的 AR 逆向换算，完美处理节点的旋转、缩放
            const localPt = node.convertToNodeSpaceAR(worldPos);
            const rect = cc.rect(
                -node.anchorX * node.width, 
                -node.anchorY * node.height, 
                node.width, 
                node.height
            );
            if (rect.contains(localPt)) {
                return node;
            }
        }
        return null;
    }

    private isValidRenderNode(node: cc.Node): boolean {
        // 这里放入你的 RenderComponent 白名单判定逻辑
        // 如：return !!node.getComponent(cc.RenderComponent); (2.x 内部判定法)
        return node.getComponent(cc.Sprite) || node.getComponent(cc.Label) /* ... */;
    }
```

## 核心模块三：跨维度折跃渲染 (The Projection Math)

这是整个方案中最具含金量的部分：**无论业务相机怎么移动、缩放，甚至被 Viewport 裁切，画出的红框都会像狗皮膏药一样死死咬住目标。**

```typescript
    // 续 InspectorCore.ts
    
    // 建议在 lateUpdate 中调用，确保拿到的是业务逻辑/动画结算后的最终坐标
    private onLateUpdate() {
        if (!this.selectedNode || !this.selectedNode.isValid || !this.targetCamera) {
            return;
        }
        this.drawGizmo(this.selectedNode, this.targetCamera);
    }

    private drawGizmo(targetNode: cc.Node, targetCamera: cc.Camera) {
        this.graphics.clear();

        // 1. 获取目标节点完美的世界坐标 OBB 四个角 (支持旋转缩放)
        const worldCorners = this.getAccurateWorldCorners(targetNode);
        if (!worldCorners) return;

        const drawPoints: cc.Vec2[] = [];

        for (const worldPt of worldCorners) {
            // 2. 空间折跃 A：目标相机世界 -> 屏幕物理视口坐标
            const screenPt = targetCamera.getWorldToScreenPoint(worldPt);
            
            // 3. 空间折跃 B：屏幕物理视口坐标 -> Inspector 相机世界坐标
            const inspectorWorldPt = this.inspectorCamera.getScreenToWorldPoint(screenPt);
            
            // 4. Inspector 世界 -> 画布局部坐标
            const localDrawPt = this.graphics.node.convertToNodeSpaceAR(inspectorWorldPt);
            
            drawPoints.push(localDrawPt);
        }

        // 5. 绘制 OBB 闭合路径
        this.graphics.moveTo(drawPoints[0].x, drawPoints[0].y);
        this.graphics.lineTo(drawPoints[1].x, drawPoints[1].y);
        this.graphics.lineTo(drawPoints[2].x, drawPoints[2].y);
        this.graphics.lineTo(drawPoints[3].x, drawPoints[3].y);
        this.graphics.close();
        
        this.graphics.stroke();
        this.graphics.fill(); // 可选：半透明高亮填充
    }

    /**
     * 获取节点真实外框的四个角点（世界坐标），完美支持 2D/3D 旋转和缩放矩阵
     */
    private getAccurateWorldCorners(node: cc.Node): cc.Vec2[] | null {
        if (node.width === 0 || node.height === 0) return null;

        const width = node.width;
        const height = node.height;
        const anchorX = node.anchorX;
        const anchorY = node.anchorY;

        // 局部四边形顶点（逆时针：左下，右下，右上，左上）
        const localPts = [
            cc.v2(-anchorX * width, -anchorY * height),
            cc.v2((1 - anchorX) * width, -anchorY * height),
            cc.v2((1 - anchorX) * width, (1 - anchorY) * height),
            cc.v2(-anchorX * width, (1 - anchorY) * height)
        ];

        const worldPts = [];
        const matrix = cc.mat4();
        // 提取节点的世界矩阵
        node.getWorldMatrix(matrix); 

        for (let i = 0; i < 4; i++) {
            const out = cc.v2();
            // 将局部点通过世界矩阵乘法，转换为真实世界坐标
            cc.Vec2.transformMat4(out, localPts[i], matrix);
            worldPts.push(out);
        }
        
        return worldPts;
    }
```

## 终极检查清单 (Checklist for Success)

当你将上述代码合入你的 `mcp-inspector-bridge` 插件后，请核对以下几点以确保万无一失：

1. [x] **是否占用了 31 号分组？** —— 必须确保目标工程没有业务逻辑写死判断 `groupIndex === 31`，一般来说 99.9% 的项目用不到这里。
2. [x] **Inspector Camera 的 clearFlags** —— 必须是 `cc.Camera.ClearFlags.DEPTH` (默认枚举值为 256)。如果是 `COLOR`，你的插件一旦启动，游戏画面就会变成黑屏或纯色背景。
3. [x] **多 Canvas 错位测试** —— 随便拉伸浏览器窗口改变比例。因为我们全链路使用的是底层 `getScreenToWorldPoint` 矩阵反算，底层代码会自动消化 Canvas 的 `FIT_HEIGHT/FIT_WIDTH` 拉伸比，红框绝不会再错位。
4. [x] **动态相机的防御** —— 业务代码如果用了 `cc.instantiate` 动态生成了预制体里的摄像机，它的默认 `cullingMask` 依然会包含 31。我们在 `onTouchEnd` 里调用的 `hijackUserCameras()` 就是为了防御这一手。

这就是目前在 Cocos Creator 2.x 引擎底层限制下，能够达到的**最顶尖、最暴力、也最优雅**的拾取架构。直接复制这份核心类，稍作生命周期绑定，你的插件将成为处理复杂场景的降维大杀器。
